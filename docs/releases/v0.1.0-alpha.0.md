# OpenHWP Studio v0.1.0-alpha.0

This is the first public alpha of OpenHWP Studio: a local-first browser workbench for Korean HWPX/HWP document workflows.

## What Works Today

- Open local `.hwpx` packages in the browser.
- Extract and edit paragraph text from `Contents/section*.xml`.
- Render existing HWPX tables as editable table cells for text-level edits.
- Preserve the source HWPX package structure when exporting edited HWPX.
- Verify exported paragraph and table-cell text by reloading the generated HWPX.
- Inspect package entries, sections, styles, relationships, manifest targets, media references, tables, and advanced-control risk.
- Run the package doctor with a health score, issue counts, repair modes, and Report JSON.
- Download a repaired HWPX for safe metadata-only repairs, currently missing root `mimetype`.
- Use bundled public synthetic samples: basic, media/BinData, and broken relationship.
- Export TXT, Markdown, HTML, JSON, HWPX, and HWP-oriented flows from the browser UI.

## Alpha Limits

- This is not a full Hancom Office replacement.
- HWP binary editing is not the primary alpha path; HWP is handled through preview/conversion-oriented flows powered by `@rhwp/core`.
- Images, drawings, headers, footers, footnotes, merged table geometry, comments, forms, and track changes need more public samples before they can be called reliable.
- CDN dependencies are still used in the alpha build.

## Why This Release Matters

Most open-source HWP/HWPX tools are engines, scripts, viewers, or editor-specific helpers. OpenHWP Studio is trying to become the browser workbench around those capabilities: inspect the package, edit the safe text surface, repair what can be repaired, and produce evidence about what changed.

## Validation

- CI checks JavaScript syntax, static DOM contracts, export formatter contracts, and HWPX fixture behavior.
- Public HWPX fixtures cover normal paragraph/table extraction, media/BinData detection, broken manifest targets, repair modes, and source-preserving patch round-trips.
- GitHub Pages deploys the static demo after successful pushes.
